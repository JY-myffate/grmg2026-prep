📐 离散表达
表达直觉理解优点缺点用途
点云一堆三维散点扫描直接出无拓扑缺面扫描原始数据

🗝 关键词速览
SDF（符号距离函数）
每个点记录到最近表面的带符号距离。负=内部，正=外部。贯穿全场的核心概念
Marching Cubes
从 SDF/体素场提取等值面为 Mesh 的经典算法——"隐式→显式"的开关
NURBS
CAD 工业标准，用控制点定义光滑曲面。非均匀有理 B 样条
UV 映射
三维顶点 → 二维纹理坐标的映射——模型"穿上衣服"的原理
半边结构 (Half-edge)
网格数据结构——每条边拆成两条有向半边，高效遍历邻域信息
BRDF
双向反射分布函数——描述材质表面如何反射光线

08 神经几何表达：几何与外观耦合

陈安沛 · 西湖大学

15:10 – 16:00
50 min · ⭐⭐⭐ 绝对核心
Day 1 最重要的课程——NeRF 和 3D Gaussian Splatting，近三年计算机图形学/视觉最具影响力的两项技术。
⚡ NeRF — 神经辐射场 (2020)
💡 直觉：想象场景内部充满发光的雾气。每个空间位置 (x,y,z) 有颜色 (RGB) 和不透明度 (σ)。一条光线穿过场景 → 沿途采样 → 累积各点的颜色×透明度 → 得到像素颜色。这个积分过程就是体渲染。
MLP 网络
输入 5D (xyz + 视角方向)，输出 4D (RGBσ)。每个场景训练一个网络
体渲染积分
沿光线路径累计颜色和透明度——"穿过半透明云层看物体的感觉"
位置编码
把坐标映射到高频空间，让 MLP 学出精细几何细节
局限
逐场景训练（小时级），渲染慢（分钟级/张），不实时
⚡ 3D Gaussian Splatting — 3DGS (2023)
💡 直觉：不再用一个大 MLP。场景由几百万个三维高斯椭球体组成，每个有位置、形状、颜色、透明度。渲染时直接投影到屏幕（像泼溅颜料一样）——实时 (>30 FPS)。
⚖️ NeRF vs 3DGS 全面对比
维度NeRF3D Gaussian Splatting
表示方式隐式 (MLP 权重)显式 (大量高斯基元)
渲染速度慢（分钟级/张）实时 (>30 FPS)
训练时间小时级分钟级
可编辑性很难（得改网络权重）可直接操作基元
几何提取需 Marching Cubes仍在研究中
应用场景学术研究、高质量重建产业落地（手机/AR/VR）
⭐ 雪宝特别提醒：这节课上完，你至少要能说清楚——NeRF 是隐式 MLP + 体渲染（慢但优雅），3DGS 是显式高斯基元 + 光栅化（快且实用）。前馈式更是一个网络看多张图直接一次出结果，不再逐个场景训练。

10 三维生成模型 (1)

🤖 四大生成模型架构
模型核心思想一句话3D 应用
VAE编码器压缩到潜空间 + 解码器重建"学会压缩-解压管道"形状插值、潜空间编辑

🔄 Diffusion 模型核心流程
干净数据 ─加噪→ 噪声数据 （网络学习预测噪声量）
随机噪声 ─去噪→ 去噪→ 去噪→ ⋯ 干净数据
🧩 SDS — 连接 2D 和 3D 的关键桥梁
💡 Score Distillation Sampling：用预训练的 2D 扩散模型（如 Stable Diffusion）作为"裁判"。3D 表示渲染出各视角的 2D 图像 → 2D 模型判断"像不像椅子" → 把信号传回 3D 表示微调。
意义：不需要 3D 训练数据，用 2D 图像就能生成 3D 模型。代表作：DreamFusion (2022)

  • ✓
    知道 Mesh、点云、体素的视觉区别 ▸

    想象你要描述一个茶壶——

    🔵 点云 🔺 网格 🧊 体素

    点云 = 茶壶表面撒上一把芝麻（⋆），每个芝麻有 (x,y,z) 坐标，但点之间没连接。

    网格 (Mesh) = 用三角形面片把点连起来，拼出表面（△▽△）。游戏/影视的工业标准格式——能渲染、能编辑、能导出。

    体素 = 像 Minecraft，把空间切成小立方体（■■■）。简单易懂，但分辨率高就内存爆炸。

    💡 口诀：点云是散芝麻，Mesh 是三角拼图，体素是乐高积木

  • ✓
    理解 SDF 是什么、Marching Cubes 干什么 ▸

    SDF（符号距离函数）：你给一个坐标 (x,y,z)，它告诉你到最近表面的距离。

    正号 = 在外部（球外 +2cm） · 零 = 在表面 · 负号 = 在内部（球内 -3cm）

    用处：布尔运算超简单（两个 SDF 取 min = 合并，取 max = 相交），生成模型喜欢用 SDF 输出。

    Marching Cubes：SDF 是隐式的（你看不到表面在哪），Marching Cubes 就是"把藏起来的表面钓出来"的算法。

    SDF 场 → Marching Cubes → 三角形网格 ✓

    💡 NeRF 里的几何提取、大多数生成模型的最后一步，都用 Marching Cubes

  • ✓
    看过 NeRF 的 3 分钟原理动画（B站搜） ▸

    实在没空看视频的话，雪宝给你讲个故事：

    想象一个房间充满半透明的彩色雾气。每个空间点 (x,y,z) 有 颜色 (RGB) 和 密度 (σ)（浓=不透明）。

    NeRF 用一个神经网络 (MLP) 来学习整个房间的"雾气地图"——

    训练时：给你看 100 张不同角度的照片 → 沿每条视线采样颜色+密度 → 积分算出像素颜色 → 和真实照片对比 → 调整网络。

    训练好了：你能从任何新角度"拍照"，得到从没拍过的视角。

    💡 为什么慢？每条光线要采样几十个点 × 每个点跑一遍 MLP，一张图要算几秒到几分钟。

    💡 3DGS 不用 MLP，改用几百万个彩色泡泡（高斯椭球体），投影渲染，实时 30 FPS。

  • ✓
    知道 Diffusion 模型的基本流程（加噪→去噪） ▸

    想象刻印章，录下雕刻过程倒放——

    训练：干净猫 ← 加噪 ← 加噪 ← ... ← 纯噪点石头
    网络学会：给定"有点噪的猫"，猜出"刚才加了什么噪"。

    生成：纯噪点石头 → 去噪 → 去噪 → ... → 干净猫
    网络每一步去掉一点噪声，走几百步，噪点就变成了清晰形状。

    三维里：随机噪声体素 → 一步步去噪 → 椅子 SDF → Marching Cubes → 椅子网格。

    💡 为什么 Diffusion 是王者？
    GAN 训练不稳定 / VAE 生成模糊 / Diffusion 质量最高、训练最稳——但生成慢（要一步步来）。
    Flow Matching 正在解决"慢"的问题。

  • ✓
    区分 显式 vs 隐式：Mesh/点云/3DGS=显式，SDF/NeRF=隐式 ▸

    全场最重要的二分法，贯穿两天讲座。

    🟦 显式："我能直接告诉你每一块在哪"
    Mesh（三角形顶点坐标）· 点云（所有点的坐标）· 3DGS（所有高斯基元的位置）
    ✅ 直观、可编辑、渲染快 · ❌ 存储大、拓扑难处理

    🟪 隐式："我不能直接告诉你表面，但能告诉你任意一点是内还是外"
    SDF（距离场）· NeRF（颜色+密度场）· MLP 网络权重
    ✅ 紧凑、支持任意拓扑（甜甜圈的洞也没问题） · ❌ 不能直接编辑

    💡 生成模型喜欢隐式（方便 AI 学），游戏/工业喜欢显式（方便渲染编辑）。
    常见 pipeline：AI 生成隐式 → Marching Cubes 转显式 → 放进游戏引擎

  • ✓
    了解 PointNet 解决了"无序点云"的问题 ▸

    问题：一个人体点云 1000 个点，

    版本 A：先列脚底的点，最后列头顶
    版本 B：先列头顶，最后列脚底

    它们代表同一个物体，只是排序不同。但普通神经网络假设输入有固定顺序——排序一变，网络以为这是另一个东西。

    PointNet 的解决方案（2017，超重要）：

    用 max pooling（对称函数）——每个点独立处理 → 所有结果取最大值
    无论点以什么顺序进来，取最大值的结果一样。

    后续：PointNet++ 加了局部结构处理，能看到椅子的腿由哪些点组成。

  • ✓
    知道 SDS 的大致概念（2D 模型指导 3D 生成） ▸

    核心困境：高质量 3D 数据很少。网上几亿张猫的图片，但只有几千个猫的 3D 模型。

    SDS（Score Distillation Sampling）：用超强的 2D 扩散模型（Stable Diffusion） 来指导 3D 生成。

    过程：
    ① 有一个"毛坯"3D 表示（NeRF 或 Mesh）
    ② 从各个角度给毛坯拍照片
    ③ 把照片扔给 Stable Diffusion："这张像真的猫吗？"
    ④ SD 说："这里应该更圆，那里有条纹"
    ⑤ 把意见传回 3D 表示，微调形状和颜色
    ⑥ 重复 → 直到各角度都像一只猫

    💡 代表作：DreamFusion (2022)，引爆了这条路线。

    局限：每个模型单独优化几分钟，细节不够。

    区分 VAE / GAN / Diffusion / Flow Matching 的直觉差异 ▸

    VAE（变分自编码器）
    "学个压缩包，解压时加点随机数" · 速度快、可插值 · ❌ 模糊

    GAN（生成对抗网络）
    "假钞犯 vs 验钞机，互相卷" · 速度快 · ❌ 训练不稳定，容易崩塌

    Diffusion（扩散模型）⭐ 当前主流
    "从噪点一步步浮现" · 质量最高、训练最稳 · ❌ 生成慢（几百步）

    Flow Matching（流匹配）🔥 最新趋势
    "Diffusion 的高速公路版" · 连续路径代替离散步骤 · ✅ 更快更稳

    💡 现在三维生成的主流顺序：传统 VAE/GAN → Diffusion 统治 → Flow Matching 崛起
